# Checklist On-Page SEO — Novos Conteúdos

## 1) Estrutura e intenção
- [ ] Página vinculada a um hub interno (institucional, serviços ou satélite).
- [ ] Objetivo da página definido (informacional, transacional ou navegação).
- [ ] Profundidade máxima de navegação: até 2 cliques a partir da home.

## 2) Semântica e headings
- [ ] Apenas 1 `<h1>` por página, com termo principal.
- [ ] Subtítulos em ordem lógica (`h2` → `h3` → `h4`), sem pular níveis.
- [ ] Seções principais com `id` e `aria-labelledby` quando aplicável.
- [ ] Blocos de conteúdo com HTML semântico (`section`, `nav`, `article`, `ul/li`).

## 3) Metadados e snippet
- [ ] `<title>` único (50–60 chars, intenção + contexto local).
- [ ] Meta description única (140–160 chars, benefício + ação).
- [ ] Canonical absoluto.
- [ ] OpenGraph/Twitter consistentes com título e descrição.

## 4) Conteúdo
- [ ] Palavra-chave principal no H1, introdução e ao menos um H2.
- [ ] Termos secundários distribuídos naturalmente no corpo.
- [ ] CTA claro com próximo passo (contato/WhatsApp/formulário).
- [ ] Prova de autoridade (experiência, credenciais, normas, método).

## 5) Links internos (hub & satélites)
- [ ] Mínimo 3 links internos contextuais por página.
- [ ] Ao menos 1 link para hub e 1 para satélite relevante.
- [ ] Âncoras descritivas (evitar "clique aqui").
- [ ] Bloco "Veja também" próximo ao fim do conteúdo.

## 6) Mídia e performance
- [ ] Imagens com `alt` descritivo e dimensão definida.
- [ ] Compressão e formato moderno (`webp/avif` quando possível).
- [ ] Sem scripts/embeds desnecessários acima da dobra.

## 7) Qualidade final
- [ ] Revisão ortográfica e consistência terminológica.
- [ ] Links testados (internos e externos).
- [ ] Validação mobile (leitura, espaçamento e CTA).
- [ ] Publicação registrada no backlog SEO/editorial.
